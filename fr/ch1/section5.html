Étude et Implémentation de LDAP Sécurisé

1.5 Le format d’échange LDIF

Le troisième aspect définit par le standard LDAP est un format d’échange de données appelé LDIF (LDAP Interchange Format ou format d’échange LDAP). Définit dans la RFC 2849 (Request For Comments), le format LDIF est un format de fichier texte normalisé permettant de stocker des informations de configuration et le contenu d’un annuaire LDAP. Il est notamment utilisé pour importer et exporter les données d’un annuaire. Un fichier LDIF est constitué :

  • d’entrées séparées par une ligne vide
  • d’associations entre des attributs et des valeurs
  • d’instructions indiquant les opérations à effectuer sur les données.

Le contenu d’un fichier LDIF est donné comme exemple plus bas.

dn: ou=Personnes, o=Entreprise
ou: Personnes
objectClass: top
objectClass: organizationalUnit

dn: uid=dupontj, ou=Personnes ,o=Entreprise
cn: Jean Dupont
sn: DUPONT
uid: dupontj
objectClass: top
objectClass: person
userPassword:: secret
Mail: jean.dupont@skynet.be
TelephoneNumber: 042049857
Description: Responsable informatique

Cet exemple décrit deux entrées. La première est une entrée de type organizationalUnit et l’autre un objet de type person. La syntaxe d’un fichier LDIF est assez simple. La première ligne d’une entrée est son DN, le reste des attributs la constituant vient à la suite. Chaque attribut d’une entrée est décrit sur une ligne repre- nant son nom et sa valeur séparée par le caractère ’ :’. Les différentes entrées contenues dans un fichier LDIF sont séparées par une ligne vide. Comme l’attribut objectclass de l’exemple, certains attributs peuvent avoir plusieurs occurrences dans une entrée.

Le format LDIF définit également des commandes permettant l’ajout, la suppression et la modification d’objets ou d’attributs. Ces commandes ne sont pas présentes dans le fichier LDIF lorsque les données sont exportées de l’annuaire (le fichier LDIF ne contient alors que des données). L’exemple ci-dessous montre le contenu d’un script LDIF (fichier LDIF contenant des instructions LDIF) permettant l’ajout d’une entrée dans l’annuaire.

dn: ou=test, o=Entreprise
Changetype: add
ou: test
objectClass: top
objectClass: organizationalUnit

La ligne "Changetype : add" indique l’opération à effectuer. Quand un fichier LDIF ne contient que des objets à ajouter, le fichier peut ne contenir que les données sans les commandes d’ajout.

dn: ou=test, o=Entreprise Changetype: modify
    add: description
    description: entree de test

dn: ou=test, o=Entreprise Changetype: modify
    replace: description
    description: modification de l’attribut description

dn: ou=test, o=Entreprise Changetype: modify
    delete: description

dn: ou=test, o=Entreprise
    Changetype: delete

Dans l’exemple présenté plus haut, l’entrée ou=test,o=Entreprise est modifiée. Dans un premier temps l’attribut description est ajouté à l’entrée (ligne 10). Ensuite la valeur de l’attribut description est mis à jour (ligne 15). Après avoir modifié l’attribut description, il est supprimé de l’entrée (ligne 20). Enfin l’entrée ou=test,o=Entreprise est également retirée de l’annuaire (ligne 24).

Les utilitaires pour l’importation ou l’exportation de données au format LDIF sont souvent fournis avec le serveur LDAP. La plupart des fournisseurs implémentent leur propre utilitaire pour l’exportation et l’importation de données au format LDIF. Par exemple, le serveur Sun iPlanet Directory offre la commande db2ldif.exe. Le serveur OpenLDAP fournit les commandes slapadd et slapcat, respectivement pour l’impor- tation et l’exportation de données.
