Étude et Implémentation de LDAP Sécurisé

2.2 Présentation d’OpenLDAP

OpenLDAP est une implémentation Open Source du protocole LDAP. Il est issu du serveur LDAP développé par l’Université du Michigan. L’ensemble du code source et de la documentation sont disponible gratuitement sur le site officiel d’OpenLDAP.

OpenLDAP est développé par une communauté d’Internautes sous le projet OpenLDAP. La première version de OpenLDAP est apparue en 1998. Depuis, OpenLDAP a beaucoup évolué. Si les versions 1.x n’implémentaient uniquement que LDAPv2, la version 2.0 de OpenLDAP3, disponible depuis 2000, est conforme au standard LDAPv3. La version actuelle est la 2.2 mais OpenLDAP 3.0 est déjà annoncée.

OpenLDAP est disponible pour la plupart des plates-formes UNIX dont GNU/Linux, Solaris, FreeBSD et Tru64. Une adaptation pour Windows est également disponible à l’adresse http://lucas.bergmans.us/hacks/openldap/.

Même s’il existe de très bons produits commerciaux 4, plusieurs arguments peuvent être avancés en faveur de OpenLDAP.

  • OpenLDAP est gratuit.
  • Le code source est accessible librement. OpenLDAP peut être adapté et modifié en fonction des besoins de l’utilisateur.
  • Sa disponibilité pour la plupart des plates-formes est un avantage considérable pour le déploiement d’un serveur LDAP dans un environnement hétérogène.

Le logiciel OpenLDAP se compose d’un serveur (slapd), d’un processus de réplication de l’annuaire (slurpd), de bibliothèques LDAP et d’utilitaires clients (ldapsearch, ldapadd...). Ces éléments sont expliqués plus en détail ci-dessous.

  • slapd (stand-alone LDAP daemon)

    est le processus serveur LDAP. Il est chargé d’écouter les connexions LDAP sur plusieurs ports (le 389 par défaut) et répond aux opérations LDAP qu’il reçoit depuis ces connexions.

  • slurpd(stand-alone LDAP update replication daemon)

    est utilisé pour la réplication de l’annuaire via le protocole LDAP. Il propage les modifications effectuées sur la base de données du serveur principal vers le serveur secondaire.

  • La bibliothèque libldap

    est une API écrite en C implémentant le protocole LDAP. Elle supporte LDAP sur TCP, LDAP sur SSL, SASL et StartTLS (voir Chapitre 1).

  • cli (command-line interface)

    Des utilitaires clients en ligne de commandes permettent d’accéder à l’annuaire. Il en existe deux types : les commandes offline (slapxxx) et onligne (ldapxxx). Les premières (slapadd, slapcat...) permettent d’administrer localement l’annuaire sans un serveur LDAP opérationnel. Les autres commandes (ldapsearch, ldapadd...) communiquent avec le serveur slapd et permettent l’accès et la mise à jour à distance de l’annuaire.

Aucune version binaire de OpenLDAP n’est proposée par le projet OpenLDAP. Pourtant, il n’est pas toujours nécessaire de compiler OpenLDAP. De nombreuses ver- sions précompilées sont fournies avec des systèmes comme Debian, Solaris ou Tru64. Mais dans les deux cas, OpenLDAP requière plusieurs logiciels externes pour le sup- port de certaines fonctionnalités. Plus précisément, OpenLDAP repose sur plusieurs bibliothèques dont :

  • Une bibliothèque de threads POSIX (en principe fournie par le système d’exploitation).
  • Des bibliothèques SSL/TLS pour la sécurité (généralement OpenSSL).
  • Une bibliothèque de gestion de base de données (typiquement la base de données Berkeley DB).
  • Des bibliothèques SASL (en général Cyrus SASL) pour le support des mécanismes d’authentification enfichables (voir Chapitre 1).
