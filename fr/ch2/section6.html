Étude et Implémentation de LDAP Sécurisé

2.6 Configuration du client Linux

Sous Linux/Debian (machine montaigne), le package à installer LDAP est ldap-utils. Ce dernier contient les utilitaires clients du logiciel OpenLDAP permettant d’accéder à un serveur LDAP local ou distant. L’utilitaire de gestion des packages apt-get peut être utilisé pour installer le package ldap-utils. La commande apt-get permet de tenir à jour le système et d’installer de nouveaux packages logiciels depuis différentes sources. Ainsi, il est possible d’obtenir les packages et les mises à jour depuis un CD-ROM, un serveur Web ou ftp. La commande apt-cache donne la liste des packages disponibles depuis les sources spécifiées dans le fichier de configuration /etc/apt/sources.list. L’exemple suivant montre comment installer le package ldap-utils.

apt-cache search "ldap" "utils"
ldap-utils - OpenLDAP utilities.

apt-get install ldap-utils
Reading Package Lists... Done
Building Dependency Tree... Done
The following NEW packages will be installed:
   ldap-utils

Une fois l’installation du package ldap-utils terminée, le client LDAP se configure au moyen du fichier ldap.conf.

2.6.1 Le fichier ldap.conf

Le fichier ldap.conf situé sous le répertoire /etc/ldap est utilisé pour définir les paramètres par défaut du client LDAP. Il doit être accessible en lecture pour tout le monde (0444). Un extrait du fichier ldap.conf est donnée comme exemple ci-dessous.

# LDAP Defaults 
#

BASE    dc=inpres, dc=be
HOST    10.59.4.8

#URI ldap://ldap.example.com ldap://ldap-master.example.com:666
#SIZELIMIT      12
#TIMELIMIT      15
#DEREF          never

Le tableau suivant décrit les principaux paramètres utilisé pour configurer le client Debian.

La valeur de l’attribut BASE définit le DN de base pour les recherches dans l’an- nuaire. L’adresse IP ou le nom du serveur LDAP à contacter est spécifié avec le para- mètre HOST.

paramètre description
BASE Spécifie le DN de base à utilisé par défaut pour les opérations LDAP.
HOST ndique la liste des serveurs à contacter. Les noms ou adresses IP doivent être séparés par deux points ( :).
PORT Numéro de port à utiliser pour se connecter au serveur.
URI Permet de spécifier l’adresse et le port du serveur sous forme d’URI10 (Uniform Resource Identifiers). Par exemple, l’URI ldap://10.59.4.8:389/ permet de contacter le serveur à l’adresse 10.59.4.8 sur le port 389.
BINDDN Indique le DN à utilisé par défaut pour s’authentifier au près du serveur. Ce paramètre est spécifique à un utilisateur et doit être enregistré dans le fichier ~ldaprc.
SIZELIMIT Nombre maximal d’entrées retournées lors des recherches.
TIMELIMIT Temps maximum à utiliser pour réaliser les recherches dans l’annuaire.

2.6.2 Tester le client

La configuration du client peut être vérifiée à l’aide de la commande ldapsearch.

montaigne:/etc/ldap# ldapsearch -x cn
version: 2

#
# filter: (objectclass=*)
# requesting: cn
#

# inpres, be
dn: dc=inpres,dc=be

# accounts, inpres, be
dn: ou=accounts,dc=inpres,dc=be

# groups, inpres, be
dn: ou=groups,dc=inpres,dc=be

# search result
search: 2
result: 0 Success

# numResponses: 4
# numEntries: 3

Le résultat de la recherche retourné par le serveur LDAP est affiché par ldapsearch dans la console. Dans l’exemple donné plus haut, ldapsearch a pu contacter le serveur et accéder aux trois entrées enregistrées dans l’annuaire.
