Étude et Implémentation de LDAP Sécurisé

4.3 Activer SSL sur le serveur LDAP

Avec le support de LDAPv3, la version 2.0 du serveur OpenLDAP est capable de mettre en oeuvre SSL/TLS. Pour cela, OpenLDAP s’appuie sur les bibliothèques partagées (shared) de OpenSSL. OpenLDAP implémente les deux mécanismes LDAPS et StartTLS. Pour rappel, LDAPS oblige l’utilisation d’un port particulier pour les sessions sécurisées (636 au lieu de 389). L’extension StartTLS (RFC 2830) permet à tout moment, au serveur et au client, de négocier des sessions SSL/TLS sur le port standard (389).

Encore une fois, l’inégalité des capacités des clients LDAP ne permet pas l’utilisation de l’extension StartTLS. Si les systèmes récents utilisant les modules PAM et NSS sont capables d’utiliser StartTLS, l’architecture SIA de Tru64 ne le permet pas. Par conséquent, le serveur OpenLDAP sera configurer avec LDAPS pour attendre les connexions SSL/TLS sur le port 636 et les connexions non sécurisées sur le port 389. La configuration du serveur OpenLDAP slapd, nécessite la mise à jour du fichier slapd.conf. OpenLDAP prévoit plusieurs directives globales. Elles sont décrites au tableau 4.2.

paramètre description
TLSCACertificateFile Spécifie le fichier contenant le certificat du CA.
TLSCACertificatePath Spécifie le chemin d’un répertoire contenant des certificats de CA placés dans des fichiers séparés.
TLSCertificateFile Spécifie le fichier contenant le certificat du serveur.
TLSCertificateKeyFile Indique le fichier contenant la clé privée du serveur associé à la clé publique contenue dans le certificat spécifié avec TLSCertificateFile.
TLSVerifyClient Spécifie le type de contrôle que le serveur doit effectuer sur le certificat éventuel du client. Les valeurs possibles sont never (le certificat du client n’est jamais réclamé), allow (si un certificat est fournit mais n’est pas valide, la session se poursuit quand même), try (si un certificat est fournit, il doit être valide), demand (le client doit fournir un certificat valide).
TLSCipherSuite Indique les modes de chiffrement acceptés et leur ordre de préférence. Il est possible de connaître les algorithmes de chiffrement supportés par OpenSSL avec la commande suivante : openssl ciphers -v ALL.

Tab. 4.2 – Paramètres SSL/TSL du fichier slapd.conf

Les modifications apportées au fichier slapd.conf sont reprises dans l’exemple suivant.

#configuration des paramètres SSL
TLSCipherSuite          HIGH:MEDIUM:+SSLv2:+SSLv3:RSA:+TLSv1
TLSCertificateFile      /data/openldap/var/ssl/slapd-cert.pem
TLSCertificateKeyFile   /data/openldap/var/ssl/slapd-key.pem
TLSCACertificateFile    /data/openldap/var/ssl/cacert.pem
TLSVerifyClient         never

Les directives TLSCertificateFile, TLSCertificateKeyFile et TLSCACertificateFile permettent au serveur de retrouver les fichiers nécessaires au fonctionnement de LDAP sur SSL (LDAPS) et StartTLS (ne sera pas implémenté pour des raisons de compatibilité). L’authentification SSL du client n’est jamais demandée (clause never de la directive TLSVerifyClient). Enfin, le serveur est configuré pour accepter les chiffrements suivants :

  • HIGH N’accepte que les clés dont la longueur ne dépasse pas 128 bits.
  • MEDIUM Accepte les clés dont la longueur est de 128 bits.
  • TLSv1, SSLv2, SSLv3 Accepte les chiffrements TLS version 1 et SSL version 2 et 3.
  • RSA Accepte les chiffrements utilisant l’algorithme RSA.

Pour que les nouveaux paramètres soient pris en compte, le serveur doit être redémarré. Pour activer LDAPS (LDAP sur SSL), le serveur doit attendre les connexions clientes sur un port particulier. Le port standard réservé à LDAPS est le port numéro 636.

LDAPS est activé au démarrage du démon slapd. L’option -h <URLs> permet de spécifier une configuration d’écoute alternative. Le configuration par défaut est ldap :///. Cette url indique l’usage du protocole LDAP au-dessus de TCP sur toutes les interfaces et sur le port par défaut du protocole LDAP (389).

Il est possible d’indiquer d’autres paires "hôte :port" et de spécifier l’usage du protocole LDAPS. Par exemple, -h "ldaps :// ldap ://127.0.0.1 :666" va demander de créer deux listeners : un pour LDAPS sur le port par défaut et un pour le protocole LDAP sur le port 666 de l’interface localhost (127.0.0.1). La commande pour lancer le serveur OpenLDAP devient :

slapd -f /usr/internet/openldap/etc/slapd.conf -d4 -h "ldaps:///"

Le serveur peut être configuré pour écouter à la fois les connexions LDAP et LDAPS.

slapd -f /usr/internet/openldap/etc/slapd.conf -d4 -h "ldap:/// ldaps:///"

Le fichier de configuration du serveur est indiqué au moyen de l’option -f. L’option -d4 demande un mode de débogage de niveau 4. Avec ce mode de débogage activé, le serveur n’est pas lancé en arrière plan et les informations de débogage sont affichés dans la console ce qui facilite les tests du serveur et des clients. Les niveaux de débogage les plus pertinents sont décris au tableau 4.3.

niveau information enregistrée
-1 ￼ Toutes les informations de débogage.
0 Aucune information de débogage.
1 Trace des appels aux fonctions.
4 Débogage verbeux.
8 Gestion des connexions.
16 Paquets réseau envoyés et reçus.

Tab. 4.3 – Niveau de débogage

Il est possible d’indiquer plusieurs niveaux en additionnant les niveaux souhaités. Par exemple, le niveau 65 active les niveaux 1 et 64.
